# Project Memories & ADR (Architectural Decision Records)

This directory serves as the "Long-term Memory" for the ZaaZ project. It is used to store the reasoning behind major architectural decisions, developmental milestones, and "contextual history" for complex logic.

---

## 📂 Directory Structure

1.  **`00x-filename.md`**: Numbered Architectural Decision Records (ADR).

---

## 🧠 Why use this?

- **Context:** Code shows _how_, Memories show _why_.
- **Onboarding:** New developers can quickly understand why certain constraints exist.
- **Legacy Avoidance:** Prevents accidental "re-solving" of problems that were already addressed.
- **Consistency:** Maintains the architectural vision over time.
